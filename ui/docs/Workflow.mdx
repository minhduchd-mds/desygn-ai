# Project Workflows

## 1. Figma Component Scan

1. Select a component or frame in Figma.
2. Open the Desygn AI plugin.
3. Click "Scan Component".
4. Review naming, structure, token, metadata, completeness, and variant scores.
5. Apply safe fixes when available.
6. Copy the compact prompt only after the score is ready.
7. Hand the prompt to Codex, Claude Code, Cursor, Windsurf, or Figma Make.

## 2. Batch Component Scan

1. Select 5-10 related components in Figma.
2. Click "Batch Scan".
3. Review atomic build order: atoms, molecules, organisms.
4. Check low-score components before exporting.
5. Copy the batch prompt.
6. Implement in dependency order and re-scan after major Figma changes.

## 3. Design.md Web Workspace

1. Open the web workspace.
2. Choose an Open Design preset or imported Design.md template.
3. Add a product request, Markdown files, ZIP files, or screenshots.
4. Generate Design.md.
5. Review Preview and Edit tabs.
6. Save edits, copy output, or download `DESIGN.md`.

## 4. BA Agent Workflow

1. Select the "BA Agent Workflow" template.
2. Import Figma context, Markdown/ZIP files, screenshots, and stakeholder notes.
3. Let the agent build project memory from confirmed facts and assumptions.
4. Review generated gaps and targeted BA questions.
5. Generate BA artifacts: `BRD.md`, `PRD.md`, `USER_STORIES.md`, `ACCEPTANCE_CRITERIA.md`, `SCREEN_FLOW.md`, and `AGENT_HANDOFF.md`.
6. Approve diffs before generated content becomes source of truth.
7. Export the handoff package for coding agents.

## 5. Design System Export

1. Open the plugin Design.md tab.
2. Refresh the design-system snapshot.
3. Map real Figma components to template sections.
4. Export a project frame when component mapping is ready.
5. Download the generated agent knowledge pack.
6. Run lint, tests, build, and visual review before marking implementation complete.

## Setting Up a Profile

1. Open profile management.
2. Create a new profile.
3. Import Figma tokens and components where available.
4. Add tech stack and implementation guidelines.
5. Save the profile.
6. Re-scan selected components with the active profile.

## Performance Tips

| Selection | Nodes | Speed |
|-----------|:-----:|-------|
| Single atom (Button) | under 100 | Instant |
| Molecule (ProductCard) | 100-500 | Fast, under 2s |
| Organism (Header) | 500-1000 | Slower, 5-10s |
| Full page | 1000+ | Not recommended |

- Scan organisms individually, never entire pages.
- Use Batch Scan to process multiple small components.
- If a scan takes more than 5s, select a smaller subtree.

## Token Cost Awareness

| Scenario | Input Tokens | Sonnet Cost |
|----------|:-----------:|:-----------:|
| Single component | 800-2,000 | ~$0.01 |
| Batch (8 components) | 10,000-18,000 | ~$0.10-0.30 |
| Batch + corrections | 20,000-36,000 | ~$0.20-0.60 |

Check the token estimate next to the Copy button before pasting.
